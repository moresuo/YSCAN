YSCAN 性能优化与现代化改造建议
================================

目标：将 YSCAN 从“脚本集合式安全工具”升级为“高性能现代化扫描器”。
核心方向：流式目标生成、有界并发调度、异步网络 I/O、成功早停、端口预探活、结构化结果输出、现代 CLI 体验。


一、当前主要性能瓶颈
--------------------

1. IP / 端口一次性展开成大列表

当前 AddressTools.py 和 PortTools.py 会一次性生成 hosts / ports 列表：

- 扫 /16 会一次性生成 65534 个 IP
- 扫 1-65535 会一次性生成 65535 个端口
- 后续 executor.submit() 会创建大量 Future 对象
- 内存和调度压力较大

建议：改为生成器，边生成目标边扫描。

示例：

def iter_segments(segment):
    if "/" in segment:
        for ip in ipaddress.ip_network(segment, strict=False).hosts():
            yield str(ip)
    elif "-" in segment:
        ip_start, ip_end = segment.replace(" ", "").split("-")
        start = int(ipaddress.ip_address(ip_start))
        end = int(ipaddress.ip_address(ip_end))
        for ip in range(start, end + 1):
            yield str(ipaddress.ip_address(ip))
    else:
        yield segment

收益：
- 降低内存占用
- 支持大网段扫描
- 更适合后续有界并发调度


二、并发模型优化
----------------

1. 当前 ThreadPoolExecutor 没有背压机制

当前很多模块都是：

for item in items:
    executor.submit(worker, item)

问题：
- threads 只限制同时执行线程数，不限制已提交任务数
- 大规模目标会排队大量 Future
- 内存压力上升

建议：分批提交或使用有界队列。

示例：

from concurrent.futures import ThreadPoolExecutor, as_completed

def run_limited(tasks, worker, threads=500, batch_size=2000):
    with ThreadPoolExecutor(max_workers=threads) as executor:
        batch = []
        for task in tasks:
            batch.append(executor.submit(worker, task))
            if len(batch) >= batch_size:
                for future in as_completed(batch):
                    future.result()
                batch.clear()

        for future in as_completed(batch):
            future.result()


2. 不同模块采用不同并发模型

推荐：

- TCP 端口扫描：asyncio.open_connection 或非阻塞 socket
- 目录扫描：httpx.AsyncClient / aiohttp
- 子域名扫描：复用 Resolver 或异步 DNS
- SSH / MySQL / Redis 爆破：线程池即可，但要控制任务队列和成功早停
- IP ping：当前 subprocess 可用，但进程开销大，后续可引入 TCP ping / ARP ping


三、TCP 端口扫描优化
--------------------

当前使用 socket + ThreadPoolExecutor，适合小规模扫描，但全端口时线程开销较大。

建议：改为 asyncio TCP connect。

示例：

import asyncio

async def scan_port(host, port, timeout=1):
    try:
        conn = asyncio.open_connection(host, port)
        reader, writer = await asyncio.wait_for(conn, timeout=timeout)
        writer.close()
        await writer.wait_closed()
        return port
    except Exception:
        return None

async def scan_ports(host, ports, concurrency=1000):
    sem = asyncio.Semaphore(concurrency)

    async def bound_scan(port):
        async with sem:
            return await scan_port(host, port)

    tasks = [bound_scan(port) for port in ports]
    for coro in asyncio.as_completed(tasks):
        result = await coro
        if result:
            print(f"[+] {host}:{result} 端口开放")

收益：
- 减少线程上下文切换
- 更适合全端口扫描
- 更接近现代高性能扫描器架构

建议新增模式：

yscan port -H 192.168.1.1 --top
yscan port -H 192.168.1.1 --full
yscan port -H 192.168.1.1 -p 22,80,443,3306,6379


四、IP 存活探测优化
-------------------

当前 ip_scan.py 已改为 subprocess.run(..., shell=False)，命令注入风险已解决。

但性能上仍有问题：每个 IP 都启动一个 ping 进程，扫大网段时开销较大。

建议支持多种探测模式：

1. ICMP ping
- 兼容性较好
- 但进程开销大
- 可能被防火墙禁用

2. TCP ping
- 探测常见端口：80, 443, 22, 445, 3389, 3306, 6379
- 如果连接成功或明确拒绝，也可判断主机在线
- 更适合内网实战

3. ARP ping
- 局域网内效率高
- 跨网段不可用
- 实现复杂度较高

建议 CLI：

yscan ip -H 192.168.1.0/24 --mode icmp
yscan ip -H 192.168.1.0/24 --mode tcp
yscan ip -H 192.168.1.0/24 --mode both
yscan ip -H 192.168.1.0/24 --mode arp


五、Web 目录扫描优化
--------------------

当前 dir_scan.py 使用 requests.get，每次请求独立连接，性能一般。

建议：

1. 使用 requests.Session

- 复用 HTTP 连接
- 在线程模型下建议每个线程一个 Session

2. 更现代的方案：使用 httpx.AsyncClient

示例：

import httpx

async def scan_dir(client, url):
    try:
        response = await client.get(url, timeout=3)
        if response.status_code in [200, 301, 302, 401, 403]:
            print(f"[+] 存在目录[{response.status_code}]：{url}")
    except Exception:
        pass

3. 增加软 404 识别

很多站点不存在路径也返回 200，需要 baseline 检测：

- 请求随机不存在路径
- 记录 status_code、content_length、title、body hash
- 扫描结果与 baseline 相似时过滤

建议参数：

yscan dir -t https://example.com --filter-soft-404

4. 增加响应长度输出

当前输出：
[+] 存在目录[200]：https://example.com/admin

建议输出：
[200] [len: 1234] https://example.com/admin
[403] [len: 567]  https://example.com/admin/

5. 支持扩展名爆破

建议参数：

yscan dir -t https://example.com -x php,asp,jsp,txt,bak,zip

候选路径生成：
admin -> admin.php / admin.jsp / admin.bak
backup -> backup.zip / backup.bak


六、子域名扫描优化
------------------

当前每扫一个子域都创建 dns.resolver.Resolver，存在额外开销。

建议：

1. 避免每个任务重复创建 Resolver
- 可复用 Resolver
- 或每个线程初始化一个 Resolver
- 或引入异步 DNS

2. 增加 wildcard DNS 检测

算法：
- 随机生成几个不存在子域，例如 yscan-random-abc.example.com
- 如果也能解析出 IP，则说明存在 wildcard DNS
- 后续扫描结果如果只命中 wildcard IP，则过滤或标记

输出示例：
[!] 检测到 wildcard DNS: *.example.com -> 1.2.3.4
[+] admin.example.com -> 1.2.3.5
[~] test.example.com -> 1.2.3.4 wildcard filtered

3. 支持多记录类型

yscan subdomain -t example.com --type A,AAAA,CNAME


七、弱口令爆破性能优化
----------------------

当前 SSH / MySQL / Redis 都是 host -> user -> password -> submit task。

1. 字典文件不要在每个 host 里重复打开

当前问题：
- 每个 host 都重新 open 密码文件
- 大字典时 I/O 重复
- 多 host 时性能下降明显

建议启动时一次性读取：

def load_lines(path):
    with open(path, encoding="utf-8") as f:
        return [line.strip() for line in f if line.strip()]

passwords = load_lines(password_path)
users = load_lines(user_path)

2. 爆破成功后停止该 host 的后续尝试

当前如果 root/123456 成功，仍会继续尝试其他密码。

建议：
- 每个 host/service 维护 found 标记
- worker 开始时检查是否已成功
- 成功后加入 found_hosts

3. 爆破前先做端口探活

流程：
1. TCP connect 判断 22/3306/6379 是否开放
2. 只对开放端口进入弱口令模块

收益：
- 大幅减少无效协议连接
- /24 中可能只有少量主机开放对应服务

建议参数：

yscan ssh -H 192.168.1.0/24 --precheck
yscan mysql -H 192.168.1.0/24 --precheck
yscan redis -H 192.168.1.0/24 --precheck

4. 凭据优先级排序

先跑高概率组合：
root/root
root/123456
root/admin
admin/admin
redis/redis
mysql/mysql

然后再跑完整用户字典 x 密码字典。


八、Redis 模块优化
------------------

当前 Redis 设计逻辑可以保留：
- 先爆破弱口令
- 弱口令成功后按 -pub / -I / -L 参数触发公钥写入或反弹 shell

建议：

1. db=10 改为 db=0

原因：部分 Redis 配置数据库数量较少，正确密码可能因为 SELECT 10 失败而漏报。

2. Redis 爆破成功后停止该 host 后续密码尝试

3. 先检测 6379 是否开放，再进入 redis-py 认证

4. 文案可更准确：
“不存在未授权访问漏洞”可改为“不满足 Redis 写入利用条件”。


九、SSH 模块优化
----------------

1. compress=True 改为 compress=False

爆破场景下压缩收益不明显，可能增加协商开销。

2. 成功连接后可不执行 echo 命令

当前成功后执行：
exec_command("echo moresuo")

connect() 成功已经可以说明凭据有效。

建议：
- 默认不执行命令，提高速度
- 如需强验证，增加 --verify-command 参数


十、MySQL 模块优化
------------------

1. 弱口令验证时不一定要指定 database="information_schema"

当前指定 database 可能在特殊权限环境下导致连接失败。

建议先只做认证连接：

conn=pymysql.connect(
    host=ip,
    port=port,
    user=user,
    password=password,
    connect_timeout=connect_timeout,
    charset="utf8",
    autocommit=True
)

成功后再查询版本：
cursor.execute("select version()")

2. 成功后停止该 host 后续尝试

3. 先检测 3306 是否开放


十一、输出性能和用户体验优化
----------------------------

当前所有模块直接在线程中 print。

问题：
- 多线程输出容易交错
- 大量 print 会拖慢扫描
- 不方便输出 JSON / CSV
- 不方便做进度条和统计

建议：

1. 引入统一 Result 对象

示例：

{
    "module": "port",
    "host": "127.0.0.1",
    "port": 80,
    "status": "open",
    "detail": ""
}

2. worker 返回结果，主线程统一输出

3. 支持输出格式：

--output result.json
--output result.csv
--json
--silent
--verbose

4. 增加最终统计

示例：

[*] 扫描完成
    目标数: 254
    任务数: 65535
    发现数: 12
    耗时: 8.43s
    平均速率: 7773 req/s


十二、CLI 现代化建议
--------------------

当前 argparse 可以继续使用。

如果要做现代化工具，建议考虑：

1. Typer

示例命令结构：

yscan port scan 192.168.1.1 --ports 1-65535
yscan web dir https://example.com --wordlist libs/dirpath.txt
yscan brute ssh 192.168.1.0/24 -u root -P passwords.txt

优点：
- 自动 help 更漂亮
- 类型提示友好
- 子命令组织清晰
- 后续容易扩展

2. Rich

用于：
- 彩色输出
- 表格
- 进度条
- 状态面板
- 更友好的异常显示

示例体验：

YSCAN v0.2
Targets      192.168.1.0/24
Mode         TCP Port Scan
Threads      500
Timeout      1s
Progress     ████████████████████ 100%


十三、架构重构建议
------------------

当前结构：

yscan.py
tools/
  xxx_scan.py
  xxx_burte.py

建议演进为：

yscan/
  cli.py
  core/
    target.py          # IP / 端口 / URL 输入解析
    scheduler.py       # 并发调度、限速、任务队列
    result.py          # 统一结果结构
    output.py          # JSON / CSV / 文本输出
  scanners/
    port.py
    ip.py
    dir.py
    subdomain.py
  brute/
    ssh.py
    mysql.py
    redis.py
  utils/
    paths.py
    net.py
    wordlist.py

好处：
- 扫描逻辑和 CLI 解耦
- 后续加新模块更容易
- 更容易写测试
- 可以插件化


十四、算法级优化总结
--------------------

1. 流式目标生成
避免一次性生成超大 hosts / ports 列表。

2. 预探活
弱口令前先判断目标端口是否开放。

3. 成功早停
发现有效凭据后停止该 host 的后续爆破。

4. 去重
对 IP、端口、路径、子域名、用户名、密码去重。

5. 批量调度
避免一次性 submit 所有任务。

6. 自适应超时
局域网：0.5s - 1s
公网：2s - 5s
Web：3s - 5s
弱口令：2s - 5s

建议支持参数：
--timeout
--retries


十五、推荐新增参数
------------------

全局参数：
-T, --threads       并发数
--timeout           超时时间

--top               常用端口

-w, --wordlist      字典文件

--filter-length     长度过滤
--soft-404          软 404 检测

-w, --wordlist      子域名字典
--dns               指定 DNS 服务器

--type              A/AAAA/CNAME

弱口令：
--precheck          先检测端口开放
--stop-on-success   成功后停止该 host
--delay             每次尝试间隔


十六、推荐落地路线
------------------

第一阶段：低成本立刻提升

1. AddressTools 改生成器，避免大网段一次性展开
2. PortTools 改生成器，避免全端口一次性列表
3. 弱口令模块字典文件只读取一次
4. 弱口令成功后停止该 host 后续尝试
5. TCP / SSH / MySQL / Redis 先做端口开放预检查
6. print 改成主线程统一输出
7. 加 --timeout、--output、--json、--quiet、--verbose

第二阶段：扫描引擎化

1. 新增 core/scheduler.py
2. 新增 core/result.py
3. 新增 core/output.py
4. 所有扫描模块返回结果，不直接 print
5. CLI 只负责解析参数和展示结果

第三阶段：异步化

1. TCP 端口扫描改 asyncio
2. 目录扫描改 httpx.AsyncClient
3. 子域名扫描支持异步 DNS
4. 保留 SSH / MySQL / Redis 线程池模式

第四阶段：现代化体验

1. Typer 重写 CLI
2. Rich 做进度条和表格
3. 支持 JSON / CSV / HTML 报告
4. 支持配置文件 yscan.yaml
5. 支持插件式 scanner


十七、最值得优先做的 10 个优化
-------------------------------

1. 弱口令字典只读取一次，不要每个 host 重复 open
2. 弱口令成功后 stop-on-success
3. 端口开放预检查后再爆破 SSH / MySQL / Redis
4. AddressTools / PortTools 改成生成器
5. TCP 端口扫描改 asyncio
6. dir_scan 改 requests.Session 或 httpx.AsyncClient
7. 子域名扫描增加 wildcard DNS 检测
8. 所有 worker 不直接 print，统一 Result 输出
9. 增加 JSON / CSV 输出和最终统计
10. 用 Rich 增加进度条和结果表格


十八、最终建议
--------------

如果目标是“高性能现代化工具”，不要只是继续堆功能，而要把 YSCAN 从脚本集合升级成：

扫描引擎 + 插件模块 + 统一结果输出 + 现代 CLI

核心能力：

- 流式生成目标
- 有界并发调度
- 异步网络 I/O
- 成功早停
- 端口预探活
- 结构化结果
- 现代化 CLI 展示

这样 YSCAN 后续可以平滑扩展到：

- 服务识别
- 指纹识别
- PoC 验证
- 资产导出
- 批量报告
- 插件化漏洞检测
